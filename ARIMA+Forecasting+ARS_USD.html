Comprobando que la serie temporal es estacionaria.¶

Que se quiere decir con comprobar si la serie temporal es estacionaria y por que nos importa?

En una serie de tiempo estacionaria, las propiedades estadísticas deben ser constantes en el tiempo y la autocovarianza debe ser dependiente del tiempo. Normalmente, cuando se corre una regresión normal, uno espera que las observaciones sean independientes entre sí. Sin embargo, en una serie temporal uno tiene certeza que las observaciones dependen del tiempo. Para poder usar técnicas de regresión en variables dependientes del tiempo la data debe ser estacionaria. Las técnicas que aplican a variables independientes aleatorias tambien aplican a variables aleatorias estacionarias.

Hay dos formas de comprobar si una serie de tiempo es estacionaria. La primera es ploear la varianza móvil y observar si se mantiene constante en el tiempo. Pero no siempre es posible hacer inferencias visuales. La otra forma es usar el test de Dickey-Fuller, un test estadístico cuya hipótesis nula es que la serie temporal es no-estacionaria. Si el test resulta en que la estadística del test es significativamente menos que el valor crítico preestablecido, podemos rechazar la hipótesis nula en favor de tomar la serie temporal como estacionaria.

Ahora calculamos la varianza móvil, ploteamos los resultados y aplicamos el test de Dickey-Fuller en la serie temporal:

def test_stationarity(timeseries):
    
    #Determing rolling statistics
    rolmean = timeseries.rolling(window=52,center=False).mean() 
    rolstd = timeseries.rolling(window=52,center=False).std()

    #Plot rolling statistics:
    orig = plt.plot(timeseries.index.to_pydatetime(), timeseries.values, color='blue',label='Original')
    mean = plt.plot(rolmean.index.to_pydatetime(), rolmean.values, color='red', label='Media móvil')
    std = plt.plot(rolstd.index.to_pydatetime(), rolstd.values, color='black', label = 'Desvío móvil')
    plt.legend(loc='best')
    plt.title('Media y desvío estandard móviles')
    plt.show(block=False)
    
    #Perform Dickey-Fuller test:
    print ('Results of Dickey-Fuller Test:')
    dftest = adfuller(timeseries, autolag='AIC')
    dfoutput = pd.Series(dftest[0:4], index=['Test Statistic','p-value','#Lags Used','Number of Observations Used'])
    for key,value in dftest[4].items():
        dfoutput['Critical Value (%s)'%key] = value
    print (dfoutput)

Estacionarizando la serie temporal¶

Si la serie de tiempo revela una tendencia o estacionalidad (no confundir estacional con estacionario), es una indicación de que es no-estacionaria. Podemos estacionarizar la serie temporal mediante calcular la tendencia y estacionalidad y remover estos factores del modelo. Se pueden aplicar diferentes transformaciones para estacionarizar la data como se verá a continuación.

Quitando tendencias y estacionalidad con diferenciación¶

La diferenciación (differencing) es uno de los métodos más comunes para tratar tanto la tendencia como la estacionalidad. En diferenciación de primer orden, se calculan las "diferencias" entre observaciones consecutivas en la serie temporal. Esto suele mejorar la estacionalidad en la serie de tiempo. En el código esto se confirma realizando el test de Dickey-Fuller.

Encontrando parámetros óptimos y construyendo el modelo ARIMA¶

Para aplicar ARIMA a la serie temporal es necesario encontrar los valores óptimos para los tres parámetros siguientes (p,d,q):

  • (p) El número de términos autoregresivos (AR): Los términos AR son simplemente desfases de la variable dependiente (el tipo de cambio en este dataset). Entonces, p=2 significa que los predictores de x(t) son x(t-1) y x(t-2).

  • (q) El número de términos de media móvil (MA) : Los términos MA son errores desfasados de predicción en la ecuación predictiva. Por ejemplo, con q=2 los predictores de x(t) son e(t-1) y e(t-2) donde e(i) es la diferencia entre la media móvil en el instante iésimo y el valor verdadero.

  • El número de diferencias (d): Estos son los números de diferencias no estacionales. En nuestro caso, d=1ya que estamos usando una serie de tiempo diferenciada en el primer orden.

Hay dos maneras de determinar el número de términos AR y MA. La primera es usar la función arma_order_select_ic de Python. La segunda utiliza los gráficos de la función de autocorrelación (ACF) y la función de autocorrelacion parcial (PACF).

En este gráfico, los valores 'p' y 'q' pueden ser determinados como sigue:

  • p: El valor donde la curva PACF corta el 0 por primera vez. (p=2)

  • q: El valor donde la curva ACF cruza el intervalo de confianza superior por primera vez. (q=2, q=1 tambien podría funcionar)

Esto significa que los valores óptimos para el modelo ARIMA(p,d,q) son (2,1,2).

Si los valores sugeridos por los gráficos de ACF y PACF difieren de los valores obtenidos por la función arma_order_select_ic, se deben probar valores para los terminos p y q y usar los resultados del modelos ajustados para obtener sus valores AIC y proceder con con el modelo que tenga un menor valor de AIC.

Haciendo predicciones en la serie temporal¶

Lo que se ha logrado en este notebook hasta aqui es una predicción dentro de la muestra usando ARIMA al entrenar el modelo en la toda ladata de la serie temporal. Ahora se necesita separar el dataset en conjuntos de training y test. Se usará el dataset de training para entrenar el modelo ARIMA y realizar una predicción fuera de la muestra. Luego se compararán los resultados de las predicciones fuera de la muestra con los valore de dataset de test.

Se usará la función "forecast" para realizar una predicción móvil de un-paso con ARIMA. Una predicción móvil esrequerida dada la dependencia en las observaciones durante la diferenciación y el modelo AR. Se re-creará el modelo ARIMA después de que cada nueva prediction sea recibida. Y manualmente se mantendrá registro de todas las observaciones en una lista llamada history, la cual tiene una semilla en la data de training y para la cual nuevas predicciones son incorporadas en cada iteración.

In [24]:
size = int(len(ts_week_log) - 15)
train, test = ts_week_log[0:size], ts_week_log[size:len(ts_week_log)]
history = [x for x in train]
predictions = list()

print('Valores predichos vs Valores Esperados')
print('\n')
for t in range(len(test)):
    model = ARIMA(history, order=(2,1,1))
    model_fit = model.fit(disp=0)
    output = model_fit.forecast()
    yhat = output[0]
    predictions.append(float(yhat))
    obs = test[t]
    history.append(obs)
    print('Predicción=%f, Esperado=%f' % (np.exp(yhat), np.exp(obs)))

error = mean_squared_error(test, predictions)

print('\n')
print('Error medio cuadrático de las predicciones')
print('MSE: %.6f' % error)

predictions_series = pd.Series(predictions, index = test.index)
